# EQ 演算法議題整理

Date: 2026-04-14

## 1. 文件目的

本文件整理目前 EQ 自適應演算法在正式落地前，必須先釐清的議題。  
目的不是直接下結論，而是整理思考脈絡，避免後續討論把不同層次的問題混在一起。

## 2. 目前我們已經有的東西

目前專案已經具備：

1. 一組可用的 preset library
2. 一個可從 preset 建立 tree 的方法
3. 一個 coarse-to-fine 的 EQ optimizer
4. 一個可模擬 emotion input 的 prototype loop

這代表我們已經能跑出一版 closed-loop prototype，但還不足以宣稱產品演算法已定稿。

## 3. 為什麼還不能直接定稿

目前最關鍵但尚未完全 specification 化的部分，是：

1. 如何評估某個 EQ 節點的情緒表現
2. 如何比較兩個節點的好壞
3. 如何根據比較結果決定 tree 應該前進、換路、撤回，還是維持

也就是說，`preset`、`tree`、`optimizer` 雖然已存在，但 `emotion-to-decision` 的規格仍未定清楚。

## 4. 必須拆開討論的三個 spec

### 4.1 Evaluation Spec

定義：

> 某一個 EQ 節點在一段時間內，情緒表現要如何量化。

需要決定：

1. emotion model 的原始輸出格式
   - binary
   - continuous score
   - probability / confidence
2. 模型最短需要多長輸入 window 才能做出有效預測
3. 每個 EQ 節點至少要觀察多久
4. 一次 evaluation 要用幾筆 emotion output

### 4.2 Decision Spec

定義：

> 當前 EQ 節點和上一個節點相比，是變好、差不多，還是變差。

需要決定：

1. 如何計算 emotion 變化
   - 單筆比較
   - 平均值比較
   - 投票
   - 加權平均
   - 差分趨勢
2. 如何把變化轉成：
   - `improve`
   - `stable`
   - `worsen`
3. threshold 如何設定
4. 是否需要 smoothing / hysteresis / confidence filter

### 4.3 Search Policy Spec

定義：

> 當知道目前 trend 之後，tree 要怎麼走。

需要決定：

1. coarse 層要先廣度搜尋還是先深度搜尋
2. medium / fine 是否要用不同策略
3. `improve` 時要走哪個 child
4. `stable` 時要換 sibling、停留、還是再觀察一次
5. `worsen` 時要回 parent、回最佳已知節點，還是回 root
6. 什麼情況下可以停止

## 5. 這些問題的依賴關係

建議討論順序：

1. `Search Policy`
2. `Evaluation Spec`
3. `Decision Spec`
4. `Stopping Criteria`

原因是：

1. 不知道樹怎麼跑，就不知道每個節點最少要嘗試多久
2. 不知道節點觀察多久，就無法定義 emotion input 的需求
3. 沒有 evaluation，就無法穩定定義 improve / stable / worsen

## 6. 目前特別值得記錄的新條件點

### 6.1 Reward 是高成本訊號

這是目前最重要的新認知之一：

> 每一次 reward 的取得，都需要使用者實際聽一段時間。

因此 reward 不是免費訊號，tree search 不能被當成可無限試探的完整搜尋問題。

這會直接導出產品限制：

1. 總測試時間必須有限
2. 單一節點的觀察次數不能太多
3. 必須允許 early stop
4. 必須思考 limited-budget search，而不是只思考最佳解

### 6.2 情緒模型能力會限制決策頻率

若模型需要例如 15 秒 PPG 才能估計 HRV，再推情緒，那就代表：

- EQ 決策頻率不能快於模型最短有效 window

因此：

1. window 長度
2. 更新頻率
3. 每個節點嘗試次數

都要由 emotion model 能力反推，而不是單獨由演算法任意設定。

### 6.3 中間節點可能也需要成為可停止點

若 reward 成本高，則不一定每次都適合追到單一 preset 再停。

需要進一步討論：

1. 是否允許停在 preset 前的中間節點
2. 什麼條件下可接受中間節點作為 stable EQ
3. 若停在中間節點，輸出的 EQ 是：
   - path delta
   - path delta + local refine
   - best-so-far node

## 7. 目前可以接受的原則

在還沒定完所有細節前，先接受以下原則：

1. emotion model 提供的是 reward signal，不是直接的 EQ 方向
2. traversal direction 必須由 reward、tree structure、policy 共同決定
3. coarse / medium / fine 很可能需要不同搜尋策略
4. reward 是高成本訊號，必須把總時間與節點預算納入考慮
5. 實驗設計要能回答：
   - 是否找到使用者喜歡的候選
   - 是否收斂夠快
   - 是否比 baseline 更有效
